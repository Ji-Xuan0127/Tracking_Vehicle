# 智能寻迹小车工程

一套基于 STM32F407 的智能寻迹小车工程，采用 STM32CubeMX + HAL 进行底层驱动开发，并结合 CMake 构建系统组织整个项目。工程围绕电机控制、传感器采集、人机显示与串口调试等能力展开，适合用于课程设计、嵌入式综合实验和智能车小项目开发。

## 项目简介

本工程的目标是实现一台能够沿轨迹线路稳定行驶的智能小车。系统以 STM32F407 为控制核心，通过电机驱动模块TB6612控制左右轮运动，利用四路寻迹模块（红外传感器）采集轨迹信息，并根据控制策略实时调整转向与速度，从而完成自动寻迹。

工程同时集成了显示与调试能力，便于在开发、调参和联调阶段快速观察系统状态，降低调试成本。

## 功能特性

- 支持智能寻迹控制，适配黑线/白底或类似轨迹场景。
- 支持左右电机独立控制，可用于差速转向与速度闭环调节。
- 集成按键输入，便于模式切换、启停控制和参数调整。
- 支持 OLED/TFT 显示，可用于展示状态、速度、传感器值或调试信息。
- 支持 ADC、DMA、I2C、SPI、UART 等外设协同工作。
- 基于 HAL 库开发，便于移植和扩展。
- 使用 CMake 组织工程，方便在不同开发环境下管理构建流程。

## 硬件平台

- 主控：STM32F407 系列 MCU
- 开发环境：STM32CubeMX 生成初始化工程，HAL 库驱动外设
- 显示模块：OLED / TFT 屏幕
- 交互模块：按键输入
- 执行机构：双电机驱动与差速控制
- 通信接口：串口调试输出
- 采集接口：ADC / DMA 等

## 软件架构

工程采用模块化组织方式，核心逻辑与外设驱动相互解耦，便于维护和扩展。

- `Core/Src/main.c`：主程序入口，负责系统初始化、主循环调度和业务逻辑衔接。
- `Core/Motor/Control.c`：电机控制相关逻辑，通常用于速度、方向和运动状态控制。
- `Core/Key/key.c`：按键扫描与按键事件处理。
- `Core/Screen/OLED`：OLED 显示驱动与字体资源。
- `Core/Screen/TFT`：TFT 显示驱动、字体与图片资源。
- `Core/Src/adc.c`：ADC 初始化与采样配置。
- `Core/Src/dma.c`：DMA 资源初始化。
- `Core/Src/i2c.c`：I2C 外设配置。
- `Core/Src/spi.c`：SPI 外设配置。
- `Core/Src/tim.c`：定时器配置，常用于 PWM、周期中断和控制节拍。
- `Core/Src/usart.c`：串口通信与调试输出。

## 开发环境

- IDE 或编辑器：VS Code、STM32CubeIDE 或其他支持 CMake 的工具链环境
- 编译器：arm-none-eabi-gcc
- 构建系统：CMake + Ninja
- 调试方式：ST-LINK / J-LINK 等 SWD 下载与调试方式

## 构建说明

项目已经带有 CMake 构建配置，可直接使用预设进行构建。

```bash
cmake --preset Debug
cmake --build --preset Debug
```

## 下载与调试

1. 使用 ST-LINK 将程序下载到 STM32F407 开发板。
2. 上电后观察串口调试输出、OLED/TFT 显示和电机动作。
3. 根据实际轨迹传感器反馈，调整阈值、控制参数和转向策略。

## 运行流程

工程的典型运行流程如下：

1. 系统上电，完成时钟、GPIO、定时器、串口和传感器初始化。
2. 读取轨迹传感器数据，判断当前车身相对轨迹的位置。
3. 根据偏差计算控制量，调整左右电机 PWM 和方向。
4. 在显示屏上刷新关键状态信息，辅助调试和观察。
5. 在按键或其他控制条件触发下完成模式切换、停止或重新启动。

## 模块说明

### 电机控制

电机控制模块负责车辆前进、转向、停止以及速度调节。电机模块同时集成了霍尔传感器，可以对电机转速进行读取便于控制。

### 轨迹采集

工程通常会通过红外循迹传感器、光电传感器或其他检测模块采集赛道信息，采集结果可用于判断左偏、右偏、居中和丢线等状态。

### 显示系统

OLED/TFT 模块用于展示运行状态、传感器数值、控制参数、通信信息等，方便在调试时快速确认系统工作情况。

### 按键与模式控制

按键模块可用于启动、停止、校准、模式切换和参数增减，让小车在不依赖上位机的情况下也能进行基本交互。

### 串口调试

串口输出适合打印传感器状态、控制变量和关键日志，是嵌入式调试阶段的重要手段。

## 注意事项

- 首次使用前，请确认时钟、引脚和外设配置与实际硬件一致。
- 若显示屏、传感器或电机驱动型号不同，需要同步修改驱动与引脚映射。
- 下载程序前，建议先确认供电、电机驱动和传感器接线正确，避免硬件损坏。
